Bekijk project
Het doel van dit eerste project van de minor was om een website te bouwen die mijzelf voorstelde en om mijn startcompetenties te overstijgen. Ik heb hiervoor een nep desktopomgeving gemaakt met hierop een terminal die mij voorstelt, oftewel een Kerrminal.

Geschreven door Kerr

Resultaat

Kerrminal is een website met de look-and-feel van een retro desktopomgeving, inclusief een werkende terminalapplicatie. De naam is een samentrekking van mijn naam (Kerr) en het woord terminal.

Mijn oorspronkelijke idee was een text-based adventure game, maar al snel besloot ik om er een interactieve terminal binnen een nep-desktopomgeving van te maken. Dat paste beter bij de eisen én was leuker om te bouwen. Het stelde me in staat om klein te beginnen en iteratief uit te breiden.

De terminal

De kern van de website is een terminal waarin de gebruiker commando's kan invoeren om informatie over mij op te vragen.

Ik heb verschillende commando's gebouwd: whoami navigeert naar een submenu met persoonlijke informatie, theme opent het thema-menu en reset zet de terminal terug naar de beginstaat.

Via het whoami-menu kunnen mijn leerdoelen worden opgevraagd met het goal-commando. Met whois is het mogelijk om via de API gegevens van een medestudent op te halen.

In het theme-menu heb ik de mogelijkheid gebouwd om de tekstkleur en vensterkleur van de terminal aan te passen en om een andere achtergrond ("wallpaper") in te stellen. De wallpapers zijn persoonlijke foto's van mij, zodat de website een persoonlijk tintje krijgt.

Login en desktopomgeving

Om de desktopervaring completer te maken heb ik een loginscherm toegevoegd. Medestudenten kunnen inloggen met hun student-ID, waarna ik via de API hun naam en profielfoto ophaal en toon in een laadscherm. Daarna komen ze terecht in de desktopomgeving.

Settings-applicatie

Naast de terminal heb ik een tweede "applicatie" gebouwd: een settings-venster waarmee de gebruiker kan wisselen tussen verschillende thema's. Elk thema stelt een persoonlijke foto van mij in als achtergrond, gekoppeld aan een van de whoami-categorieën. Ik heb een stuk of tien thema's gemaakt, waaronder een standaard licht en donker thema. De settings-app gaf de desktopomgeving ook meteen wat meer diepte, doordat er nu twee applicaties naast elkaar konden draaien.

Wat ik heb geleerd

Mijn belangrijkste leerdoel was om minder perfectionistisch te zijn en frequenter iets werkends op te leveren. Ik heb mezelf uitgedaagd om klein te beginnen en van daaruit stap voor stap uit te breiden, in plaats van één groot ambitieus doel na te streven. Dat is me goed gelukt.

Iets anders dat ik heb geleerd is werken met CSS-animaties en transitions. Ik had daar voorafgaand aan dit project nog nooit mee gewerkt. Ik heb geleerd hoe ik transitions toepas en hoe ik scroll-driven animaties opzet.

Om de wallpaper dynamisch te wisselen heb ik voor het eerst CSS-properties rechtstreeks vanuit JavaScript gemanipuleerd. Dit opende voor mij nieuwe mogelijkheden voor interactieve stijling.

Ik heb ook geleerd hoe ik meerdere thema's kan inladen in een website en hoe ik op basis van de voorkeur van de gebruiker automatisch het juiste thema activeer.

Als laatste moest de ASCII-art in de terminal op verschillende schermformaten goed werken. Ik heb geleerd hoe ik dit met media queries dynamisch en netjes kon oplossen.
