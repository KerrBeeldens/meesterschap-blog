The Rule of Least Power

De achterliggende reden is dat HTML en CSS declaratieve talen zijn. Je beschrijft wat je wilt bereiken, terwijl de browser bepaalt hoe dat gebeurt. JavaScript is daarentegen imperatief: je schrijft stap voor stap voor wat er moet gebeuren. Declaratieve oplossingen zijn daardoor vaak eenvoudiger, minder fragiel en beter onderhoudbaar.

Als het met HTML kan, gebruik dan geen CSS. Als het met CSS kan, gebruik dan geen JavaScript. — Kilian Valkhof

HTML en CSS kan meer dan je denkt

Het moderne webplatform biedt bovendien steeds meer mogelijkheden waarvoor vroeger JavaScript nodig was. Enkele voorbeelden:

  • Sliders als checkboxes: met HTML en CSS kunnen traditionele selectievakjes worden omgevormd tot moderne schuifknoppen, zonder extra scripting.
  • Smooth scrolling: vloeiend scrollen naar een sectie kan rechtstreeks via CSS. Daarbij is het belangrijk rekening te houden met gebruikers die bewegingsanimaties willen beperken via prefers-reduced-motion. Met scroll-margin kan worden voorkomen dat een doelanker achter een vaste header terechtkomt. De :target pseudo-klasse maakt het vervolgens mogelijk om de aangeklikte sectie extra te benadrukken.
  • Accordions: uitklapbare inhoud kan eenvoudig worden gerealiseerd met de HTML-elementen details en summary. Zelfs de standaard marker kan worden aangepast met CSS.
  • Dialogs en pop-ups: het dialog-element biedt ingebouwde ondersteuning voor modale vensters. Via ::backdrop kan de achtergrond worden gestileerd, terwijl @starting-style helpt om soepele animaties toe te voegen.
  • Automatisch meegroeiende tekstvelden: dankzij field sizing kan een tekstvak zich aanpassen aan de hoeveelheid ingevoerde tekst, zonder JavaScript.
  • Aangepaste selectievakjes: met appearance: base-select krijgen ontwikkelaars meer controle over de vormgeving van keuzelijsten, terwijl de standaardfunctionaliteit behouden blijft.

Deze voorbeelden tonen aan dat moderne browsers steeds meer functionaliteit rechtstreeks aanbieden. Door eerst naar HTML en CSS te kijken en JavaScript pas als laatste redmiddel te gebruiken, ontstaan oplossingen die eenvoudiger, robuuster en beter aansluiten bij de filosofie van het web.

Wat neem ik hiervan mee?

De belangrijkste les voor mij is om bewuster na te denken over de technologie die ik inzet. In de praktijk grijp ik soms snel naar JavaScript om interactie toe te voegen, terwijl het moderne webplatform veel van deze functionaliteit inmiddels standaard aanbiedt via HTML en CSS. Door eerst te onderzoeken wat mogelijk is met de ingebouwde mogelijkheden van de browser, kan ik vaak eenvoudiger en robuuster bouwen.

Daarnaast heeft deze presentatie me laten inzien hoeveel het webplatform de afgelopen jaren is geëvolueerd. Functionaliteiten zoals accordions, dialogs, smooth scrolling en dynamische formulierelementen vereisen vaak geen extra JavaScript meer. Dat zorgt niet alleen voor minder code, maar ook voor betere toegankelijkheid en onderhoudbaarheid.

Wat is de minst krachtige technologie die dit probleem kan oplossen?

Dit is denk ik een waardevolle vraag om mezelf te stellen als ik aan een probleem werk. Pas wanneer HTML en CSS tekortschieten, zal JavaScript mijn volgende stap zijn.
