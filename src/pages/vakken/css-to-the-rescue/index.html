CSS to the Rescue

Bekijk project
Bij dit vak was de uitdaging om een interactieve website te bouwen met uitsluitend HTML en CSS. Zonder JavaScript, en zonder gebruik te maken van klassen of ID's, tenzij strikt noodzakelijk voor het koppelen van een label aan een input. De nadruk lag daarmee volledig op het creatief inzetten van CSS als interactietaal, iets waar ik me tijdens dit project echt in heb moeten verdiepen.

Geschreven door Kerr

Resultaat

Ik heb een interactieve tutorial gemaakt waarmee gebruikers stap voor stap leren hoe ze een Rubik's Cube kunnen oplossen: Solve It NOW!. Het project was allesbehalve een rechte lijn van idee naar eindproduct. Ik heb in totaal drie repositories, omdat ik halverwege het project opnieuw ben begonnen met de kennis die ik tot dan toe had opgedaan. Dat was een moeilijke maar goede beslissing.

Eerste pogingen

Voordat ik aan de echte interface begon, heb ik uitgebreid geëxperimenteerd in een apart side project. Ik wilde begrijpen hoe ik een 3D Rubik's Cube kon bouwen en laten draaien met puur CSS. De kubus bestaat uit 27 div-elementen met een slimme structuur: een buitenste wrapper, daarbinnen een div per blokje met inline custom properties voor de positie, en daarbinnen zes divs voor de zes zijden.

De rotatie werkte via radio buttons die onzichtbaar zijn gemaakt, waarbij het label als knop diende. Door te kijken welke checkbox required was en aangevinkt, paste ik via CSS-selectoren de juiste rotatie toe op de juiste blokjes. De selectors werden al snel complex, maar het werkte.

Het echte probleem was de schaal. Een Rubik's Cube oplossen kost zo'n 120 moves over 27 blokjes, en voor elke move moest ik per blokje bepalen of het meedraaide, over welke as, en in welke richting. Dat met de hand uitwerken was nagenoeg onmogelijk.

CSS genereren met Java

Om dit probleem op te lossen, bouwde ik een apart hulpprogramma in Java. Dit programma leest een bestand met moves in, simuleert de kubus intern per blokje, houdt de positie en lokale rotatie bij na elke stap, en genereert vervolgens automatisch de benodigde HTML en CSS. Zo hoefde ik de complexe transformaties niet met de hand uit te rekenen, maar kon ik me richten op de structuur en logica van het hoofdproject.

Het project zelf

Terwijl ik experimenteerde in het side project en het hulpprogramma bouwde, werkte ik parallel aan de eigenlijke interface. Ik begon met een ruwe level select en een basis voor het level scherm, maar was daar niet tevreden over. In het weekend heb ik dit omgezet in een definitieve versie.

Het titelscherm bestaat uit kubussen die voor elke letter van de titel om hun as draaien. Eén voor één stoppen ze met draaien, totdat SOLVE IT NOW volledig zichtbaar is. Naast de titel ziet de gebruiker een kubus waarop de beschikbare levels verschijnen. Als een level wordt gehaald, lost de kubus zich voor een deel op en wordt het volgende level zichtbaar, totdat alle levels zijn gehaald en de kubus volledig is opgelost.

In het level scherm zelf krijgt de gebruiker een korte uitleg via een carousel, waarna hij de juiste moves moet uitvoeren. De kubus kan vrij worden rondgedraaid via knoppen onderin. Zodra alle stappen van een level zijn doorlopen, keert de gebruiker terug naar het hoofdscherm met een afsluitende animatie.

Wat ik heb geleerd

Dit project was het moeilijkste van de minor. Ik heb meerdere keren een doodlopende weg gevonden, opnieuw moeten beginnen en keuzes moeten maken over wat wel en niet haalbaar was binnen de tijd. Maar juist daardoor heb ik hier veel van geleerd.

Het meest waardevolle inzicht was hoe ver je CSS als interactietaal kunt rekken. Door radio buttons en checkboxes te combineren met geavanceerde selectors zoals :has(), ~ en :not(), bouwde ik een soort state machine die de toestand van het programma bijhield. Dat is een manier van denken die ik voor dit project nooit had overwogen.

Tegelijkertijd leerde ik de waarde van het bouwen van hulpmiddelen. Het idee om een Java-programma te schrijven om CSS en HTML te genereren voelde in het begin misschien omslachtig, maar het bespaarde me enorm veel tijd en fouten.

Verder heb ik tijdens dit project voor het eerst gewerkt met subgrid, CSS if(), oklch als kleurruimte en SVG's inline in CSS via background-image. Elk van die technieken had zijn eigen leercurve, maar samen gaven ze me het gevoel dat ik CSS nu op een heel ander niveau begrijp dan aan het begin van de minor.
