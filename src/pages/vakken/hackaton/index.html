Hackathon

Bekijk project
De hackathon was de eerste keer dat ik voor het bouwen van een website in een groepje heb gewerkt. In een korte, intensieve sprint werkten we samen aan één gezamenlijk product.

Geschreven door Kerr

Resultaat

Voordat we ook maar één regel code schreven, hebben we eerst de tijd genomen om het eens te worden over hoe we wilden samenwerken. We begonnen met een Miro board waarop we konden brainstormen over de richting van het project. We zochten inspiratie op en verkenden welke informatie we hadden gekregen en wat we daarmee wilden doen.

Uiteindelijk kozen we ervoor om ons te richten op het menselijke aspect van het bedrijf: het feit dat het vrijwel volledig door studenten wordt beheerd. Later voegden we daar ook de bedrijven aan toe die het project mogelijk maken.

Met dat idee als vertrekpunt maakten we een ontwerp in Figma, zodat we allemaal hetzelfde beeld hadden van wat we wilden bouwen.

Op basis van de verschillende elementen die we wilden implementeren verdeelden we het werk via GitHub Projects. De code conventies stelden we ook op in Miro, grotendeels gebaseerd op de conventies van FDND, aangevuld met een aantal afspraken die we zelf expliciet wilden vastleggen.

Als groep werkten we met een main, dev en feature branch structuur, aangevuld met pull requests om de branches netjes te houden. Merge conflicts losten we soms met wat gedoe op, maar we hebben het project weten af te ronden zonder grote problemen.

Mijn bijdrage

Mijn eigen bijdrage was het bouwen van een zwart gat dat studenten opslokt.

Tijdens het vak CSS to the Rescue had ik geleerd dat je elementen een SVG-pad kunt laten volgen met offset-path. Dat heb ik hier toegepast om afbeeldingen en andere elementen langs logaritmische spiralen te laten bewegen, waarbij elke spiraal net een andere fase heeft. Daardoor lijken ze in elkaar te draaien en ontstaat het effect van een zwart gat dat alles naar binnen trekt. Naast mijn eigen onderdeel heb ik anderen in het team geholpen waar nodig en heb ik veel van de pull requests afgehandeld.

Wat ik heb geleerd

Samenwerken aan een codebase was voor mij nieuw, en ik merkte al snel hoe belangrijk het is om vooraf goede afspraken te maken. Niet alleen over wie wat doet, maar ook over hoe je de code schrijft en hoe je Git gebruikt. De combinatie van GitHub Projects, een duidelijke branch strategie en pull requests zorgde ervoor dat we ondanks de korte tijdspanne gestructureerd konden werken. Het was soms even puzzelen met merge conflicts, maar het gaf me ook een goed gevoel voor hoe samenwerking in een professionele setting eruit kan zien.

Technisch gezien was het fijn om offset-path opnieuw toe te passen, maar nu in een complexere context. Het uitwerken van logaritmische spiralen met verschillende fasen vroeg om meer wiskundig en visueel redeneren dan ik gewend was. Dat ik het effect uiteindelijk werkend had gekregen gaf me veel voldoening.
